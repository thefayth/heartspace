# Public / Private Boundary

## Public

- Project purpose.
- Public-safe visuals.
- Workflow concepts.
- Ownership policy.
- WordPress page draft.
- Status and roadmap.

## Private

- Source code.
- Prompts and agent instructions.
- Credentials and secrets.
- Deployment systems.
- Databases and user records.
- Account exports.
- Operational logs.
- Private product strategy and unpublished creative work.

## Rule

The public repository should make the project understandable without exposing
the private engine.
